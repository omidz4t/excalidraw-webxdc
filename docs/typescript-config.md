# TypeScript Configuration

## Root `tsconfig.json`

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "lib": ["dom", "dom.iterable", "esnext"],
    "strict": true,
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "noEmit": true,
    "isolatedModules": true,
    "resolveJsonModule": true,
    "baseUrl": ".",
    "paths": { /* @excalidraw/* aliases */ }
  },
  "include": ["packages", "excalidraw-app"],
  "exclude": ["examples", "dist", "types", "tests"]
}
```

### Path aliases

All `@excalidraw/*` packages resolve to **source files**, not `dist/`:

```
@excalidraw/excalidraw     → packages/excalidraw/index.tsx
@excalidraw/element        → packages/element/src/index.ts
@excalidraw/common         → packages/common/src/index.ts
@excalidraw/math           → packages/math/src/index.ts
@excalidraw/utils          → packages/utils/src/index.ts
@excalidraw/fractional-indexing → packages/fractional-indexing/src/index.ts
@excalidraw/laser-pointer  → packages/laser-pointer/src/index.ts
```

Wildcard subpaths also mapped (e.g. `@excalidraw/element/*` → `packages/element/src/*`).

### Type checking

```bash
bun run test:typecheck   # runs `tsc` with noEmit
```

## Per-package configs

Each package has its own `tsconfig.json` extending `packages/tsconfig.base.json`:

| Package | Emits types to |
| --- | --- |
| common | `dist/types/common/src/` |
| element | `dist/types/element/src/` |
| math | `dist/types/math/src/` |
| utils | `dist/types/utils/src/` |
| excalidraw | `dist/types/excalidraw/` |

Generated by `bun run gen:types` (= `tsc`) after esbuild.

## Type import conventions (0.18.x)

Use `import type` from subpaths:

```ts
// Correct
import type { ExcalidrawElement } from "@excalidraw/element/types";
import type { AppState } from "@excalidraw/excalidraw/types";

// Deprecated (removed in 0.18.x)
import type { ... } from "@excalidraw/excalidraw/types/element/types.js";
```

## Strict mode implications

- `strict: true` — all strict checks enabled
- No implicit `any`
- Null checks enforced
- Path aliases must match exactly (case-sensitive on Linux)

## Vite / Vitest alias parity

Both `vite.config.mts` and `vitest.config.mts` define identical `@excalidraw/*` aliases to source. This ensures tests run against source, not built packages.
