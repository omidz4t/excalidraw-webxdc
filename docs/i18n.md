# Internationalization (i18n)

Excalidraw supports 50+ languages via JSON locale files.

## Architecture

```
packages/excalidraw/i18n.ts       ← Language detection, useI18n hook
packages/excalidraw/locales/      ← Translation JSON files
packages/excalidraw/locales/en.json          ← English (fallback)
packages/excalidraw/locales/percentages.json ← Translation completion %
```

## Usage in code

```ts
import { t, useI18n, defaultLang, languages } from "@excalidraw/excalidraw";

// In components:
const { t } = useI18n();
return <button>{t("buttons.clearCanvas")}</button>;

// Outside React:
import { t } from "@excalidraw/excalidraw/i18n";
```

## Language list

`languages` array in `i18n.ts` contains metadata per language:

```ts
type Language = {
  code: string;        // e.g. "de-DE"
  label: string;       // "Deutsch"
  rtl?: boolean;       // Right-to-left
};
```

## Lazy loading

Standard app build splits locales into separate chunks:

```ts
// vite.config.mts manualChunks
if (id.includes("locales/") && !en.json && !percentages.json) {
  return `locales/${locale}`;
}
```

Only English loads initially; other languages load on demand.

## Language detection

`excalidraw-app/app-language/`:

- `language-detector.ts` — browser language detection
- `language-state.ts` — Jotai atom for selected language
- `LanguageList.tsx` — language picker UI

Uses `i18next-browser-languagedetector` in the app shell.

## Translation keys

Keys are nested JSON paths:

```json
{
  "buttons": {
    "clearCanvas": "Clear canvas",
    "save": "Save"
  },
  "alerts": {
    "invalidEncryptionKey": "Invalid encryption key"
  }
}
```

Accessed as `t("buttons.clearCanvas")`.

## Contributing translations

Upstream uses [Crowdin](https://crowdin.com) for community translations. See [Excalidraw contributing guide](https://docs.excalidraw.com/docs/introduction/contributing#translating).

## WebXDC: English only

The WebXDC slim build hardcodes English:

1. `vite-slim-plugin` replaces dynamic locale import with `fallbackLangData`
2. `languages` array becomes `[defaultLang]`
3. All locale chunks excluded from bundle
4. `WebxdcApp` sets `langCode="en"`

This saves significant bundle size (~50 locale JS files).
